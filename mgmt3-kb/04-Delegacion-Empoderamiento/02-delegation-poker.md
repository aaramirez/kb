## Delegation Poker

Delegation Poker es una herramienta interactiva de Management 3.0 diseñada para **hacer explícitas las expectativas de delegación** dentro de un equipo. Crea una conversación estructurada sobre quién debería tener cuánta autoridad para diferentes tipos de decisiones.

> **"Lo que no se discute explícitamente, se asume — y las suposiciones generan conflictos."**

## Materiales necesarios

- **Cartas de niveles**: 7 cartas numeradas del 1 al 7 (o tarjetas/Post-its escritos)
- **Temas de decisión**: Una lista de decisiones relevantes para el equipo
- **Participantes**: Todo el equipo + manager(s)
- **Tiempo**: 60-90 minutos para una sesión completa

Si no tienes cartas físicas, puedes usar dedos (levantar 1-7 dedos), sticky notes con números, o cualquier material discrecional.

## Las 7 cartas

| Carta | Nivel | Descripción corta |
|-------|-------|-------------------|
| 1 | Tell | Yo decido y te informo |
| 2 | Sell | Yo decido y te explico por qué |
| 3 | Consult | Yo decido, pero escucho tu opinión primero |
| 4 | Agree | Decidimos juntos, necesitamos acuerdo |
| 5 | Advise | Tú decides, yo te aconsejo |
| 6 | Inquire | Tú investigas y me recomiendas |
| 7 | Delegate | Tú decides completamente |

## Preparación (10 min)

1. **Definir los dominios de decisión**: Escribe en tarjetas separadas los tipos de decisiones que el equipo enfrenta regularmente. Ejemplos:
   - Cómo dividir el trabajo entre miembros
   - Elegir herramientas técnicas
   - Definir horarios de trabajo
   - Priorizar features del producto
   - Decidir arquitectura de software
   - Establecer estándares de código

2. **Repartir las cartas**: Cada participante recibe un set de cartas del 1 al 7.

3. **Recordar los niveles**: Repasa brevemente los [[01-siete-niveles-autoridad|siete niveles de autoridad]] para que todos estén alineados.

## Gameplay (30-50 min)

### Ronda de revelación

1. El facilitador lee un tema de decisión en voz alta: *"¿Quién decide cómo se distribuye el trabajo en el equipo?"*
2. Cada participante selecciona en secreto la carta que representa el nivel de delegación que considera adecuado.
3. **Reveal simultáneo**: Todos revelan sus cartas al mismo tiempo. Esto evita el sesgo de anclaje.
4. Se observan las cartas: ¿Hay consenso o hay disparidad?

### Ronda de discusión

Si las cartas son diferentes (lo cual es lo más habitual y deseable):

1. **Los de nivel más bajo explican primero**: "¿Por qué crees que el manager debería decidir esto?"
2. **Los de nivel más alto explican después**: "¿Por qué crees que el equipo debería tener esta autoridad?"
3. **Búsqueda de acuerdo**: La meta no es forzar consenso sino llegar a un nivel que todos puedan respaldar.
4. **Re-votación**: Después de la discusión, nueva ronda de cartas. Repetir si hay discrepancia significativa.

### Documentar

Registra el nivel acordado para cada tema. Estos resultados se usan para construir el [[03-delegation-board|Delegation Board]].

## Preguntas guía para la discusión

- ¿Qué información tiene el grupo que podría no estar considerando?
- ¿Quién tiene la competencia necesaria para esta decisión?
- ¿Cuál es el costo de equivocarse? ¿Es reversible?
- ¿Qué nivel nos permitiría aprender sin generar riesgo innecesario?
- ¿Hay razones regulatorias o legales que limiten la delegación?

## Errores comunes

1. **Manager vota primero**: Si el manager revela su carta primero, el equipo tiende a alinearse. Siempre reveal simultáneo.
2. **Buscar consenso para todo**: El nivel 4 (Agree) no es siempre el correcto. A veces el nivel 3 (Consult) con una decisión clara es más eficiente.
3. **Hacer una sola sesión**: Las primeras sesiones generan resistencia. Repetir con diferentes dominios construye la práctica.
4. **Ignorar los resultados**: Si el juego genera acuerdos pero nunca se implementan, la herramienta pierde credibilidad rápidamente.

## Cuándo usar Delegation Poker

- **Equipos nuevos**: Para establecer expectativas desde el inicio sobre cómo se tomarán las decisiones
- **Transiciones de liderazgo**: Cuando cambia el manager o se reorganiza el equipo
- **Conflictos de autoría**: Cuando hay ambigüedad sobre quién decide qué
- **Retrospectivas de delegación**: Para calibrar periódicamente si los niveles de delegación siguen siendo adecuados
- **Onboarding**: Para integrar nuevos miembros y alinear expectativas

## Resultados esperados

Al final de una sesión exitosa, el equipo tendrá:

- Un **mapa claro** de niveles de delegación por dominio de decisión
- **Acuerdos explícitos** que reemplazan suposiciones tácitas
- Una **base para conversaciones futuras** cuando los niveles necesiten ajustarse
- Mayor **confianza mutua** al haber expuesto y discutido expectativas

Para una visualización permanente de los resultados, construye un [[03-delegation-board]].
