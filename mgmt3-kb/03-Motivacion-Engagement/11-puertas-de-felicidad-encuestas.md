# Puertas de Felicidad y Encuestas

## Introducción

Management 3.0 ofrece múltiples mecanismos para recolectar feedback sobre la satisfacción del equipo. Cada uno tiene un propósito, frecuencia y nivel de formalidad diferente. Este artículo compara las opciones disponibles y ayuda a elegir la adecuada según el contexto.

La clave no es usar todas las herramientas sino **elegir la combinación correcta** para las necesidades del equipo en un momento dado. Demasiadas encuestas generan fatiga; muy pocas generan ceguera organizacional.

## Espectro de herramientas

```
Informal ←————————————————————→ Formal
Bajo esfuerzo                    Alto esfuerzo
Alta frecuencia                  Baja frecuencia

Happiness Door → Pulse Check → Happiness Index → 12 Preguntas
```

## 1. Happiness Door (Puerta de Felicidad)

**Naturaleza:** Check-in continuo, informal
**Frecuencia:** Semanal o por evento
**Esfuerzo:** Muy bajo (30 segundos)
**Anonimato:** Semi-anónimo

**Ideal para:** Detectar cambios rápidos en el ánimo del equipo, especially después de eventos significativos.

**Ver:** [[06-happiness-door]] para detalles completos.

## 2. Pulse Checks (Chequeos de pulso)

**Naturaleza:** Check-in breve, semi-formal
**Frecuencia:** Semanal o quincenal
**Esfuerzo:** Bajo (2-3 minutos)
**Anonimato:** Variable

**Formato típico:**
1. Del 1 al 10, ¿cómo estás? (felicidad general)
2. Del 1 al 10, ¿cómo va el proyecto? (confianza en el proyecto)
3. ¿Hay algo que te preocupe? (campo abierto, opcional)

**Ideal para:** Monitoreo regular sin sobrecarga administrativa.

**Ejemplo en Slack:**
```
📢 Pulse Check — Semana 28

1. ¿Cómo estás? (1-10)
2. ¿Cómo va el proyecto? (1-10)
3. ¿Alguna preocupación? (opcional)

Responde con una reacción o en el hilo:
```

## 3. Happiness Index (Índice de Felicidad)

**Naturaleza:** Encuesta periódica, formal
**Frecuencia:** Mensual o trimestral
**Esfuerzo:** Medio (5 minutos)
**Anonimato:** Totalmente anónimo

**Ideal para:** Tracking de tendencias a largo plazo, análisis profundo de satisfacción.

**Ver:** [[07-happiness-index]] para detalles completos.

## 4. Encuestas formales

**Naturaleza:** Evaluación completa, muy formal
**Frecuencia:** Semestral o anual
**Esfuerzo:** Alto (15-30 minutos)
**Anonimato:** Totalmente anónimo

**Componentes típicos:**
- Satisfacción general (escala Likert)
- satisfacción con el manager
- satisfacción con el equipo
- satisfacción con las herramientas y procesos
- satisfacción con las oportunidades de crecimiento
- NPS interno (¿recomendarías este equipo?)
- Preguntas abiertas sobre mejoras

**Ideal para:** Evaluación organizacional, benchmarking temporal, decisiones estratégicas de RRHH.

## 5. Las 12 Preguntas Importantes

**Naturaleza:** Evaluación personal profunda, anual
**Frecuencia:** Anual
**Esfuerzo:** Alto (30-60 minutos de reflexión)
**Anonimato:** Semi-anónimo (se comparte con el manager)

**Ideal para:** Conversaciones 1-on-1 significativas, alinear expectativas individuales con el equipo.

**Ver:** [[14-12-preguntas-importantes]] para detalles completos.

## Matriz de decisión

| Situación | Herramienta recomendada |
|-----------|------------------------|
| Cambio reciente en el equipo | Happiness Door |
| Check-in semanal del equipo | Pulse Check |
| Tracking mensual de tendencias | Happiness Index |
| Evaluación anual de satisfacción | Encuesta formal + 12 Preguntas |
| Equipo nuevo (primeros 3 meses) | Happiness Door + Pulse Check |
| Equipo en crisis | Happiness Door + Conversación grupal |
| Preparación para retrospectiva profunda | Happiness Index |

## Frecuencias recomendadas

### Para equipos estables
```
Semanal:    Happiness Door (opcional)
Quincenal:  Pulse Check
Mensual:    Happiness Index
Anual:      12 Preguntas + Encuesta formal
```

### Para equipos en cambio
```
Diaria:     Happiness Door
Semanal:    Pulse Check
Mensual:    Happiness Index
Trimestral: 12 Preguntas (si es necesario)
```

### Para equipos remotos
```
Semanal:    Pulse Check digital
Quincenal:  Happiness Door ( Slack/Teams)
Mensual:    Happiness Index
Anual:      12 Preguntas ( video)
```

## Combatiendo la fatiga de encuesta

La fatiga de encuesta es el enemigo número uno de estos sistemas. Señales de fatiga:
- Tasa de respuesta cayendo
- Respuestas cada vez más cortas y genéricas
- Comentarios negativos sobre "tantas encuestas"
- Ignorancia de los resultados

**Estrategias de mitigación:**

1. **Alterna herramientas** — No uses la misma todo el tiempo
2. **Mantén las encuestas cortas** — Máximo 3-5 preguntas
3. **Comunica las acciones** — Si la gente ve que su feedback importa, mantiene la participación
4. **Celebra la participación** — Reconoce que dar feedback es un acto de confianza
5. **Respeta el "no"** — Si alguien no quiere participar, respétalo
6. **Reduce frecuencia** — Si la fatiga es alta, reduce la frecuencia temporalmente

## Análisis de resultados

### Para herramientas de bajo esfuerzo (Happiness Door, Pulse Check)
- Observa tendencias semanales
- Identifica outliers (alguien consistentemente bajo)
- Busca correlaciones con eventos del equipo

### Para herramientas de medio esfuerzo (Happiness Index)
- Tracking mensual con gráficas
- Análisis de dispersión (¿el equipo está alineado?)
- Correlación con métricas de productividad

### Para herramientas de alto esfuerzo (Encuestas formales, 12 Preguntas)
- Análisis cualitativo profundo de respuestas abiertas
- Comparación con períodos anteriores
- Segmentación por rol, antigüedad, etc.
- Planes de acción concretos y seguimiento

## Errores comunes

- **Usar todas las herramientas a la vez** — Overkill, genera fatiga
- **No actuar sobre los resultados** — La encuesta sin acción destruye la confianza
- **No ser consistente** — Si cambias la frecuencia o formato constantemente, la gente se confunde
- **Publicar datos identificables** — El anonimato es crucial para la honestidad
- **Hacer encuestas demasiado largas** — Nadie completa un cuestionario de 50 preguntas
