# Happiness Door

## Descripción

La Happiness Door es una herramienta simple pero poderosa de Management 3.0 para recolectar retroalimentación continua sobre la satisfacción del equipo. Consiste en una "puerta" física o virtual donde los miembros del equipo pueden dejar su nivel de felicidad de forma rápida y anónima.

A diferencia de las encuestas formales, la Happiness Door está diseñada para ser de bajo esfuerzo y alta frecuencia. No reemplaza las encuestas sino que complementa la retroalimentación continua con un mecanismo de señal rápida.

## Cuándo usarla

- **Después de una retrospectiva** — Para medir si la retrospectativa fue útil
- **Al final de un sprint** — Como check-in rápido del equipo
- **Después de un cambio significativo** — Nuevo proceso, nueva herramienta, reorganización
- **Periódicamente** — Como parte de la rutina del equipo (semanal o quincenal)
- **Después de un evento** — Workshop, presentación, capacitación

## Materiales necesarios

### Formato físico
- **Un panel o pizarra** en un lugar visible del equipo
- **Post-its de diferentes colores** (o un sistema de colores definido)
- **Etiquetas** para las categorías
- **Un indicador de "puerta"** (puede ser literalmente un dibujo de una puerta o un espacio designado)

### Formato digital
- **Slack/Microsoft Teams** — Canal dedicado o integración con encuesta
- **Miro/Mural** — Tablero con zona de felicidad
- **Google Forms** — Formulario simple de 2 preguntas
- **Herramienta Management 3.0** — La plataforma oficial

## Setup paso a paso

### Paso 1: Define la escala
Elige un sistema de medición consistente:
- **Opción A: Emoji** 😊 😐 😞 (simple, visual)
- **Opción B: Números 1-5** (1=muy infeliz, 5=muy feliz)
- **Opción C: Colores** (Verde=Amarillo=Rojo)
- **Opción D: Caras** (Sonriente=Neutral=Triste)

### Paso 2: Crea el espacio
Físico: Coloca un panel visible en el área del equipo. Digital: Crea un canal o tablero dedicado.

### Paso 3: Establece la rutina
Decide la frecuencia y momenta exacta:
- **Semanal:** Cada viernes después del standup
- **Por sprint:** Al final de cada sprint
- **Ad-hoc:** Después de eventos significativos
- **Mensual:** Como check-in general del equipo

### Paso 4: Anuncia y demuestra
Explica a qué es la Happiness Door y cómo funciona. El facilitador debe ir primero para demostrar que es seguro y anónimo.

## Cómo funciona en la práctica

### Ejemplo de setup físico
```
┌─────────────────────────────────────────┐
│           HAPPINESS DOOR                │
│                                         │
│  😊        😐        😞                 │
│ Muy       Regular    Infeliz            │
│ feliz                                      │
│                                         │
│  ┌─────┐  ┌─────┐  ┌─────┐             │
│  │ AD  │  │ ML  │  │ JP  │  ← Post-its │
│  │ 😊  │  │ 😐  │  │ 😊  │             │
│  └─────┘  └─────┘  └─────┘             │
│                                         │
│  "¿Cómo te sientes hoy?"               │
└─────────────────────────────────────────┘
```

### Ejemplo de ejecución
1. **Viernes 10:00 AM** — El facilitador dice: "Es hora de la Happiness Door"
2. Cada persona toma un post-it y escribe sus iniciales + emoji/nivel
3. Lo coloca en la zona correspondiente
4. Opcionalmente, escribe una palabra o frase sobre por qué
5. El facilitador observa los resultados y, si hay rojos, agenda una conversación

## Análisis de resultados

### Indicadores a observar
- **Tendencia temporal** — ¿La felicidad sube, baja o se mantiene?
- **Distribución** — ¿Todos están en la misma zona o hay dispersión?
- **Outliers** — ¿Alguien está consistentemente rojo? (requiere atención)
- **Correlaciones** — ¿Hay eventos que coinciden con cambios en la felicidad?

### Acciones según el resultado

| Resultado | Acción sugerida |
|-----------|-----------------|
| Mayoría verde | Celebrar, entender qué funciona bien |
| Mezcla verde-amarillo | Conversar sobre los amarillos, identificar mejoras |
| Mayoría roja | Acción inmediata: retro dedicada al problema |
| Un solo rojo | Conversación 1-on-1 para entender la situación |
| Tendencia a la baja | Investigar causas raíz, posible cambio sistémico |

## Seguimiento

La Happiness Door solo tiene valor si se actúa sobre los resultados. Pasos de seguimiento:

1. **Revisar** los resultados después de cada uso
2. **Identificar** patrones o problemas recurrentes
3. **Conversar** con las personas afectadas (especialmente si hay rojos)
4. **Actuar** sobre los hallazgos en las retrospectivas
5. **Comunicar** las acciones tomadas al equipo
6. **Repetir** para verificar si las acciones tuvieron efecto

## Errores comunes

- **No hacer nada con los resultados** — La gente deja de participar si no ve cambios
- **Forzar la explicación** — La Happiness Door debe ser de bajo esfuerzo
- **Confundir con encuesta** — No es un reemplazo para feedback profundo
- **Hacerla demasiado frecuente** — Frecuencia diaria puede generar fatiga
- **No respetar el anonimato** — Si la gente siente que está siendo observada, no será honesta

## Happiness Door vs Happiness Index

| Aspecto | Happiness Door | Happiness Index |
|---------|---------------|-----------------|
| Frecuencia | Continua | Periódica (mensual/trimestral) |
| Esfuerzo | Muy bajo | Medio |
| Anonimato | Semi-anónimo | Totalmente anónimo |
| Profundidad | Superficial | Moderada |
| Uso | Check-in rápido | Análisis de tendencia |
| Seguimiento | Inmediato | Planificado |

Ambas herramientas son complementarias. La Happiness Door para el pulso constante, el Happiness Index para el análisis profundo.
