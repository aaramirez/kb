## Change Management Game

El Change Management Game es una herramienta de Management 3.0 diseñada para que los equipos **experimenten con diferentes enfoques de cambio organizacional** en un entorno seguro y controlado. En lugar de debatir teóricamente sobre cómo gestionar el cambio, el juego permite vivir las consecuencias de distintas estrategias de manera experencial.

> **"La mejor manera de aprender sobre el cambio es experimentándolo, no leyendo sobre ello."**

## Cuándo usarlo

- **Equipos que enfrentan un cambio significativo** — reorganización, nuevo proceso, nueva herramienta
- **Managers que quieren entender su estilo de cambio** — para identificar fortalezas y áreas de mejora
- **Equipos que han resistido cambios anteriores** — para explorar por qué y qué podría funcionar mejor
- **Workshops de liderazgo** — como actividad central para desarrollar habilidades de cambio
- **Retrospectivas** — cuando el equipo necesita renovar su enfoque de mejora

## Materiales necesarios

- **Fichas de escenarios** (ver abajo) — al menos 8-10 escenarios diferentes
- **Fichas de estrategias** — tarjetas con diferentes enfoques de cambio
- **Dados o generador de números** — para seleccionar escenarios aleatorios
- **Pizarra o tablero** — para registrar resultados
- **Temporizador** — para las rondas de discusión
- **Papel y marcadores** — para notas de reflexión

## Setup paso a paso

### Paso 1: Prepara el espacio (10 min)

Organiza el espacio para que los participantes puedan trabajar en equipos pequeños (3-5 personas). Cada equipo necesita una mesa, fichas de escenarios, fichas de estrategias y material de registro.

### Paso 2: Forma los equipos (5 min)

Divide a los participantes en equipos de 3-5 personas. Si el grupo es pequeño, puede trabajar como un solo equipo. Asegúrate de que cada equipo tenga una mezcla de roles si es posible.

### Paso 3: Presenta las reglas (10 min)

Explica el flujo del juego:
1. Se revela un **escenario de cambio** (el reto)
2. Cada equipo elige una **estrategia de cambio** para abordarlo
3. Se simula la **ejecución** con consecuencias
4. Se discuten los **resultados** y se registra el aprendizaje
5. Se repite con un nuevo escenario y estrategia

### Paso 4: Distribuye las fichas (5 min)

Reparte a cada equipo las fichas de escenarios boca abajo y las fichas de estrategias boca arriba para que puedan elegir.

## Fichas de escenarios

Cada escenario describe una situación de cambio que el equipo debe abordar:

| # | Escenario | Contexto |
|---|-----------|----------|
| 1 | **Nuevo proceso de code review** | El equipo no tiene estándares claros. Los PRs tardan días en revisarse. |
| 2 | **Migración a microservicios** | La arquitectura monolítica no escala. Hay resistencia por la complejidad. |
| 3 | **Adopción de Scrum** | Otro equipo quiere adoptar Scrum pero nadie ha trabajado con Agile antes. |
| 4 | **Nuevo horario de oficina** | La empresa quiere implementar trabajo híbrido 3+2. Hay opiniones encontradas. |
| 5 | **Reorganización de equipos** | Los equipos se van a reestructurar por dominio en lugar de por función. |
| 6 | **Herramienta de tracking obligatoria** | La dirección quiere implementar métricas de productividad. Hay desconfianza. |
| 7 | **Reducción de reuniones** | El equipo se reúne demasiado. Se propone eliminar el 50% de las reuniones. |
| 8 | **Cultura de documentación** | Poca documentación genera dependencias. Se quiere cambiar la cultura. |
| 9 | **Onboarding mejorado** | Los nuevos tardan 3 meses en ser productivos. Se necesita un proceso mejor. |
| 10 | **Retrospectivas diferentes** | Las retrospectivas se han vuelto monótonas. Se necesita innovar el formato. |

## Fichas de estrategias

Cada estrategia representa un enfoque diferente para gestionar el cambio:

| Estrategia | Descripción | Nivel de involucramiento |
|-----------|-------------|--------------------------|
| **Mandato** | La dirección decide y comunica. Se espera cumplimiento. | Bajo |
| **Demostrar** | Se hace un piloto y se muestran resultados antes de expandir. | Medio |
| **Co-crear** | Se involucra al equipo desde el diseño hasta la implementación. | Alto |
| **Consultar** | Se pide opinión antes de decidir. La decisión final la tiene otro. | Medio |
| **Incentivar** | Se ofrecen recompensas para quienes adopten el cambio. | Bajo-Medio |
| **Educando** | Se invierte en formación y capacitación antes del cambio. | Medio |
| **Experimentando** | Se prueban pequeñas versiones del cambio y se itera. | Alto |

## Cómo jugar

### Ronda 1: Primer intento (15 min)

1. **Revela el escenario** — El facilitador lee el escenario en voz alta
2. **Elige la estrategia** — Cada equipo selecciona la estrategia que usaría (puede discutir internamente)
3. **Simula la ejecución** — El facilitador genera consecuencias basadas en una tabla de resultados
4. **Registra el resultado** — Cada equipo anota qué funcionó y qué no

### Tabla de consecuencias (simplificada)

El facilitador usa esta matriz para guiar las consecuencias:

| Estrategia → Escenario | Escenario de bajo riesgo | Escenario de alto riesgo |
|------------------------|--------------------------|--------------------------|
| Mandato | Adoptado pero con resistencia silenciosa | Rechazo activo o sabotaje |
| Demostrar | Adopción gradual con entusiasmo | Lentitud pero aprendizaje |
| Co-crear | Adopción entusiasta | Posible parálisis por análisis |
| Consultar | Mejora moderada | Sensación de "preguntaste pero no hiciste caso" |
| Incentivar | Adopción rápida pero superficial | Dependencia de incentivos |
| Educando | Adopción sólida pero lenta | Retención a largo plazo |
| Experimentando | Aprendizaje continuo | Incertidumbre pero datos reales |

### Ronda 2: Segundo intento (15 min)

1. Se revela un **nuevo escenario** (o el mismo para comparar)
2. Los equipos pueden **cambiar de estrategia** o quedarse con la misma
3. Se discute: ¿qué aprendieron de la ronda anterior?
4. Se registran los nuevos resultados

### Ronda 3: Discusión (15 min)

1. **Compartir hallazgos** — Cada equipo comparte sus resultados
2. **Identificar patrones** — ¿Qué estrategias funcionaron mejor en qué contextos?
3. **Reflexionar** — ¿Cómo se relaciona esto con los cambios reales del equipo?
4. **Comprometerse** — ¿Qué estrategia usarán para el próximo cambio real?

## Debrief

El debrief es la parte más valiosa del juego. Guía la conversación con estas preguntas:

1. **¿Qué sorprendió?** — ¿Alguna consecuencia fue inesperada?
2. **¿Qué patrones viste?** — ¿Las mismas estrategias funcionaron igual en diferentes escenarios?
3. **¿Cómo se compara con tu experiencia real?** — ¿Has visto estas dinámicas en tu trabajo?
4. **¿Qué harías diferente?** — Si pudieras volver a empezar un cambio real, ¿qué estrategia usarías?
5. **¿Qué necesitas para experimentar?** — ¿Qué pequeño cambio podrías probar esta semana?

## Errores comunes

1. **Saltarse el debrief** — El juego sin reflexión es solo entretenimiento
2. **No cambiar de estrategia entre rondas** — El punto es experimentar con diferentes enfoques
3. **Tomarlo demasiado en serio** — Es una simulación, no una predicción exacta
4. **No conectar con la realidad** — Siempre cerrar con "¿y en nuestro equipo?"
5. **No registrar aprendizajes** — Si no se documenta, se olvida

## Variantes

- **Versión remota**: Usa Miro o Mural para las fichas y tableros
- **Versión rápida**: Solo 2 rondas + debrief de 10 min
- **Versión profunda**: Agregar una ronda donde se diseña un plan real de cambio
- **Versión individual**: Cada persona juega sola contra un oponente
