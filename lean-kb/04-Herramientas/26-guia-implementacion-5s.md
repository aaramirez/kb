# Guía de Implementación 5S

Esta guía paso a paso describe cómo implementar [[01-5s|5S]] en un área de trabajo, desde la planificación hasta la auditoría sostenida. La implementación típica toma **4-6 semanas** para un área piloto.

## Fase 1: Preparación (Semana 1)

### 1.1 Seleccionar Área Piloto
Elegir un área que sea:
- **Visible** — Que otros departamentos puedan ver el resultado
- **Acotada** — No demasiado grande para empezar
- **Con problemas claros** — Donde 5S tenga impacto medible
- **Con un líder comprometido** — Alguien que lidere el cambio

### 1.2 Formar el Equipo
- **Líder 5S:** Responsable de coordinar la implementación
- **Equipo:** 4-6 personas que trabajan en el área
- **Patrocinador:** Un miembro de dirección que apoye

### 1.3 Capacitación Inicial
Entrenar al equipo en:
- Conceptos de las 5S
- El propósito de cada S
- Cómo se mide el éxito
- El cronograma de implementación

### 1.4 Fotos del Antes
Tomar fotografías del estado actual del área. Son la línea base para comparar después.

## Fase 2: Clasificar - Seiri (Semana 2)

### 2.1 Inspección de Objetos
Revisar **cada objeto** en el área y clasificarlo:

| Categoría | Acción |
|-----------|--------|
| Necesario y usado frecuentemente | Mantener en el área |
| Necesario y usado infrequentemente | Mover a almacenamiento cercano |
| No necesario | Retirar del área |

### 2.2 Zona Roja
Marcar los objetos no necesarios con etiquetas rojas. Dar un plazo (3-5 días) para que las personas reclamen lo que necesitan. Lo no reclamado se retira.

### 2.3 Medir Resultados
Contar cuántos objetos se retiraron y cuánto espacio se liberó.

## Fase 3: Ordenar - Seiton (Semana 3)

### 3.1 Definir Lugares
Para cada objeto necesario, asignar un lugar específico:
- **Frecuencia de uso:** Lo más usado, más cerca
- **Punto de uso:** Donde se necesita, ahí se guarda
- **Peso/tamaño:** Lo pesado a nivel waist, lo liviano arriba

### 3.2 Etiquetar y Marcar
- Etiquetas claras y legibles
- Siluetas de herramientas (shadow boards)
- Marcas de piso para contenedores
- Códigos de colores por categoría

### 3.3 Estándar de Orden
Crear un estándar visual que muestre cómo debe verse el área ordenada. Usar fotos del "estado ideal" como referencia.

## Fase 4: Limpiar - Seiso (Semana 4)

### 4.1 Limpieza Profunda
Realizar una limpieza completa del área (no solo superficial):
- Limpiar debajo y detrás de máquinas
- Limpiar herramientas y equipos
- Reparar fugas y daños detectados

### 4.2 Inspección Durante Limpieza
La limpieza es oportunidad para detectar:
- Fugas de aceite o agua
- Desgaste en componentes
- Daños en herramientas
- Condiciones inseguras

### 4.3 Establecer Rutinas
Crear un calendario de limpieza:
- **Diario:** Limpieza básica del área de trabajo
- **Semanal:** Limpieza profunda de equipos
- **Mensual:** Inspección completa del área

## Fase 5: Estandarizar - Seiketsu (Semana 5)

### 5.1 Documentar Estándares
Crear documentos visuales:
- **Fotos del estado ideal** del área
- **Checklist de limpieza** con responsables
- **Mapa del área** con ubicaciones de cada cosa
- **Procedimientos escritos** para cada rutina

### 5.2 Asignar Responsabilidades
Cada persona debe saber:
- Qué área le corresponde
- Qué rutina debe cumplir
- Con qué frecuencia
- A quién reportar problemas

### 5.3 Tablero de 5S
Crear un tablero visual que muestre:
- Asignaciones de área
- Calendario de limpieza
- Resultados de auditorías
- Fotos del estado ideal

## Fase 6: Sostener - Shitsuke (Semana 6+)

### 6.1 Auditorías Regulares
Establecer un calendario de auditorías:
- **Frecuencia:** Al menos quincenal al inicio
- **Equipo:** Mezclar auditores de dentro y fuera del área
- **Checklist:** Lista de verificación estandarizada
- **Puntuación:** Sistema numérico para medir progreso

### 6.2 Checklist de Auditoría Tipo

| Criterio | Puntuación (1-5) |
|----------|-------------------|
| Clasificación correcta (no hay objetos innecesarios) | |
| Orden correcto (cada cosa en su lugar) | |
| Limpieza (área limpia y sin contaminación) | |
| Estandar visible (checklist y fotos actualizados) | |
| Disciplina (el equipo sigue las rutinas) | |

### 6.3 Reconocimiento
- Celebrar mejoras en puntaje de auditoría
- Reconocer responsables de áreas destacadas
- Compartir resultados con la organización

## Errores Comunes en la Implementación

| Error | Consecuencia | Solución |
|-------|-------------|----------|
| Saltar la capacitación | El equipo no entiende el propósito | Invertir tiempo en enseñar |
| Hacerlo solo con "limpieza" | Se pierde el enfoque en eficiencia | Enfocarse en las 5S completas |
| No hacer auditorías | Shitsuke falla, las S se degradan | Programar auditorías obligatorias |
| No involucrar al equipo | Resistencia al cambio | Incluir a los operadores desde el inicio |
| Implementar muy rápido | Fatiga y rechazo | Seguir el cronograma de 4-6 semanas |

## Métricas de Éxito

| Métrica | Antes | Después (target) |
|---------|-------|-------------------|
| Tiempo de búsqueda | [MEDIR] | Reducir 50%+ |
| Espacio liberado | [MEDIR] | 20%+ del área |
| Puntuación auditoría | [MEDIR] | 80%+ en 3 meses |
| Accidentes/incidentes | [MEDIR] | Reducir a cero |
