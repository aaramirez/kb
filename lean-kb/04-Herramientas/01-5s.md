# 5S - Organización del Espacio de Trabajo

Las **5S** son una metodología de [[Lean Manufacturing]] para organizar el espacio de trabajo de forma que sea seguro, eficiente y productivo. Originada en Japón, las 5S constituyen la base visual sobre la cual se construyen todas las demás herramientas Lean.

## Origen

El sistema 5S fue desarrollado en Toyota como parte del [[01-Fundamentos/01-que-es-lean|Sistema de Producción Toyota (TPS)]] bajo la guía de [[01-Fundamentos/01-historia-lean|Taiichi Ohno]]. Originalmente eran 4S, y se añadió la quinta S con el tiempo.

## Las Cinco S

### 1. Seiri (整理) — Clasificar

Separar lo necesario de lo innecesario. Es el acto de revisar todo lo que hay en el área de trabajo y retirar lo que no se necesita.

**Acciones clave:**
- Identificar todos los objetos en el área de trabajo
- Clasificar cada objeto como necesario o innecesario
- Retirar los objetos innecesarios del área
- Decidir dónde almacenar o disponer de lo retirado

**Preguntas guía:**
- ¿Cuándo fue la última vez que usé este objeto?
- ¿Lo necesito realmente para mi trabajo diario?
- ¿Hay otra persona que lo necesite?

### 2. Seiton (正頓) — Ordenar

Organizar los objetos necesarios de manera que se puedan encontrar y usar rápidamente. Todo tiene un lugar designado y todo está en su lugar.

**Acciones clave:**
- Asignar un lugar específico para cada objeto
- Organizar por frecuencia de uso (lo más usado, más accesible)
- Etiquetar y marcar ubicaciones
- Utilizar siluetas, contenedores y sistemas visuales

**Principio:** "Un lugar para cada cosa y cada cosa en su lugar."

### 3. Seiso (清掃) — Limpiar

Mantener el área de trabajo limpia y ordenada. La limpieza no solo es estética, sino que permite detectar anomalías tempranamente.

**Acciones clave:**
- Establecer estándares de limpieza
- Asignar responsabilidades de limpieza
- Limpiar como forma de inspección (detectar fugas, desgaste, daños)
- Crear rutinas de limpieza diarias

**Relación con [[06-gemba|Gemba]]:** La limpieza permite una mejor observación durante los paseos Gemba.

### 4. Seiketsu (清潔) — Estandarizar

Crear estándares y reglas para mantener las primeras tres S. Estandarizar los procedimientos para que cualquiera pueda mantener el área organizada.

**Acciones clave:**
- Crear procedimientos escritos y visuales
- Definir estándares de limpieza y organización
- Establecer auditorías periódicas
- Usar listas de verificación y fotografías del estado ideal

**Conexión con [[10-pdca|PDCA]]:** Los estándares se crean (Plan), se implementan (Do), se verifican (Check) y se ajustan (Act).

### 5. Shitsuke (躾) — Sostener

Desarrollar la disciplina y los hábitos para mantener las 4S anteriores. Es la S más difícil porque requiere cambio cultural.

**Acciones clave:**
- Realizar auditorías regulares
- Reconocer y celebrar el progreso
- Capacitar continuamente
- Liderar con el ejemplo desde la dirección

**Relación con [[02-Pilares/03-kaizen|Kaizen]]:** Shitsuke es la base para que la mejora continua sea sostenible.

## Beneficios de 5S

| Beneficio | Impacto |
|-----------|---------|
| Reducción de tiempo de búsqueda | Hasta 60% menos tiempo buscando herramientas |
| Menos defectos | Menos contaminación, daños y errores por desorden |
| Mayor seguridad | Menos accidentes por obstáculos y condiciones inseguras |
| Mejor moral del equipo | Espacios agradables generan orgullo y compromiso |
| Base para otras herramientas | Sin 5S, Kanban, Andon y Flujo Continuo no funcionan |
| Detección temprana de problemas | La limpieza revela fugas, desgaste y anomalías |

## Pasos de Implementación

1. **Selección del área piloto** — Elegir un área visible y acotada
2. **Formar equipo** — Incluir a quienes trabajan en el área
3. **Seiri (Clasificar)** — Retirar lo innecesario, crear zona "roja"
4. **Seiton (Ordenar)** — Ubicar cada cosa en su lugar
5. **Seiso (Limpiar)** — Limpieza profunda y establecer rutinas
6. **Seiketsu (Estandarizar)** — Documentar estándares con fotos
7. **Shitsuke (Sostener)** — Auditorías y mejora continua
8. **Expandir** — Replicar a otras áreas tras el éxito piloto

## 5S y Gestión Visual

Las 5S son la base de la [[02-andon|gestión visual]] en Lean. Un espacio 5S permite:
- Identificar anomalías rápidamente (un objeto fuera de lugar es visible)
- Comunicar el estado del área sin palabras
- Facilitar los paseos [[06-gemba|Gemba]]
- Soportar sistemas [[09-kanban|Kanban]] físicos

## 5S en Software

Las 5S también se aplican al desarrollo de software:
- **Seiri:** Eliminar código muerto, dependencias obsoletas
- **Seiton:** Organizar archivos, estructura de directorios clara
- **Seiso:** Refactoring, limpiar technical debt
- **Seiketsu:** Code standards, linting rules
- **Shitsuke:** Code reviews, CI/CD pipelines

## Errores Comunes

- Implementar 5S como "limpieza forzada" sin compromiso del equipo
- No mantener las auditorías (Shitsuke falla)
- Enfocarse solo en lo cosmético, no en la eficiencia
- No involucrar a los trabajadores en la clasificación
