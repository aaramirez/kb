# Metodología DMAIC

## ¿Qué es DMAIC?

DMAIC es la metodología central de [[05-Vertientes/5b-Lean-Six-Sigma/01-lean-six-sigma-vista-general|Lean Six Sigma]]. Es un ciclo de 5 fases para resolver problemas de procesos de forma sistemática y basada en datos.

> *"DMAIC no es un proyecto. Es una forma de pensar para resolver problemas de raíz."*

## Las 5 fases

```
Define → Measure → Analyze → Improve → Control
  ①        ②         ③         ④         ⑤
```

### Fase 1 — Define ([05-Vertientes/5b-Lean-Six-Sigma/03-dmaic-definir])
- ¿Qué problema estamos resolviendo?
- ¿Por qué importa al negocio?
- ¿Quién es el cliente afectado?
- **Entregable:** Project Charter

### Fase 2 — Measure ([05-Vertientes/5b-Lean-Six-Sigma/04-dmaic-medir])
- ¿Cuál es el estado actual con datos?
- ¿Qué tan bueno/malo es el proceso?
- ¿Nuestros datos son confiables?
- **Entregable:** Análisis de capability

### Fase 3 — Analyze ([05-Vertientes/5b-Lean-Six-Sigma/05-dmaic-analizar])
- ¿Cuáles son las causas raíz?
- ¿Qué variables impactan más el resultado?
- ¿Hay patrones en los datos?
- **Entregable:** Análisis de causas raíz

### Fase 4 — Improve ([05-Vertientes/5b-Lean-Six-Sigma/06-dmaic-mejorar])
- ¿Qué soluciones eliminan las causas raíz?
- ¿Funcionan en la práctica?
- ¿Cuál es el impacto esperado?
- **Entregable:** Plan de implementación

### Fase 5 — Control ([05-Vertientes/5b-Lean-Six-Sigma/07-dmaic-controlar])
- ¿Cómo mantenemos las mejoras?
- ¿Qué monitoreamos?
- ¿Quién es responsable?
- **Entregable:** Control Plan

## Cuándo usar DMAIC

| Situación | ¿Usar DMAIC? |
|-----------|-------------|
| Problema repetitivo con datos disponibles | Sí |
| Proceso nuevo sin datos | No — usar DMADV |
| Mejora incremental de un proceso existente | Sí |
| Diseño de un producto nuevo | No — usar DMADV |
| Problema ambiguo sin causa identificada | Sí — excelencia para causas raíz |
