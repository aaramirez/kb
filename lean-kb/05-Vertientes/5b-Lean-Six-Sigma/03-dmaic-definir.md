# DMAIC: Definir

## El propósito de Define

La fase Define establece **qué problema vamos a resolver**, por qué importa y qué éxito esperamos. Sin una buena definición, todo el proyecto falla.

> *"Un problema bien definido está medio resuelto."* — Charles Kettering

## Project Charter

El Project Charter es el documento fundacional del proyecto:

| Elemento | Descripción | Ejemplo |
|----------|-------------|---------|
| **Problema** | Descripción clara del issue | "El 15% de pedidos llegan con errores" |
| **Objetivo** | Meta medible | "Reducir a <2% en 6 meses" |
| **Alcance** | Qué incluye y qué no | "Solo pedidos del canal B2B" |
| **Business case** | Impacto financiero | "$500K/año en costos de retrabajo" |
| **Equipo** | Roles y responsabilidades | GB, BB, sponsor, procesos owner |
| **Timeline** | Fechas de cada fase | 4-6 meses típicamente |

## Voice of the Customer (VOC)

El VOC captura lo que el cliente **realmente necesita**:

- **Requisitos críticos** (CTQs — Critical to Quality)
- **Encuestas y entrevistas**
- **Quejas y reclamos**
- **Observación directa**
- **Análisis de datos de mercado**

### CTQ Tree
```
Necesidad del cliente → Driver → requisito medible
  "Entrega a tiempo"  → "Planificación" → "98% on-time delivery"
```

## SIPOC

El SIPOC es un diagrama de alto nivel del proceso:

| Elemento | Definición | Ejemplo |
|----------|-----------|---------|
| **S**upplier | Quién provee inputs | Proveedor A, B |
| **I**nput | Qué entra al proceso | Materia prima, datos |
| **P**rocess | Pasos principales del proceso | Recibir → Procesar → Empacar |
| **O**utput | Qué sale del proceso | Producto terminado |
| **C**ustomer | Quién recibe el output | Cliente final |

## Herramientas complementarias

- **Stakeholder Analysis** — quién afecta y es afectado por el proyecto
- **High-Level Process Map** — mapa de 5-8 pasos del proceso
- **Project Selection** — matrices Pugh o prioritización por impacto/esfuerzo

## Errores comunes en Define

- Definir el problema demasiado vago ("mejorar calidad")
- No tener datos de VOC reales
- Alcance demasiado amplio (proyecto de 12+ meses)
- Falta de sponsor comprometido
- Objetivos no medibles
