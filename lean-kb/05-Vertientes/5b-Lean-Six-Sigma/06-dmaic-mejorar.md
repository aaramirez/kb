# DMAIC: Mejorar

## El propósito de Improve

La fase Improve desarrolla, prueba e implementa **soluciones que eliminan las causas raíz** identificadas en Analyze. No es adivinar — es experimentar de forma controlada.

> *"Mejorar no es implementar la primera idea. Es probar la mejoridea con datos."*

## Desarrollo de soluciones

### Generación de soluciones
- **Brainstorming** del equipo
- **Benchmarking** — qué hacen otros
- **Analogías** — soluciones de otros procesos
- **Método de los 7 pasos creativos**
- **TRIZ** — resolución de contradicciones

### Selección de soluciones
| Criterio | Herramienta |
|----------|-------------|
| Impacto en el problema | Matriz de selección |
| Facilidad de implementación | Matriz de impacto/esfuerzo |
| Costo-beneficio | Análisis financiero |
| Riesgo | FMEA (Failure Mode and Effects Analysis) |

## Pilot testing

Antes de implementar a escala completa:

1. **Diseñar el piloto** — definir alcance, duración, métricas
2. **Ejecutar controlada** — mismo proceso, solo cambia la solución
3. **Medir resultados** — comparar con baseline de [[05-Vertientes/5b-Lean-Six-Sigma/04-dmaic-medir|Measure]]
4. **Validar** — ¿cumple el objetivo del proyecto?
5. **Ajustar** — refinar la solución si es necesario
6. **Escalar** — implementar en todo el proceso

### Fórmula de validación
```
Resultado piloto vs. Baseline = ¿Se cumple el objetivo?
  Si Sí → proceder a implementar
  Si No → iterar o buscar otra solución
```

## Implementación

### Plan de implementación
| Elemento | Descripción |
|----------|-------------|
| **Qué** | Solución específica a implementar |
| **Quién** | Responsable de cada acción |
| **Cuándo** | Cronograma con hitos |
| **Dónde** | Áreas de implementación |
| **Cómo** | Pasos detallados |
| **Riesgos** | Contingencias |

### Gestión del cambio
- Comunicación clara del porqué
- Involucrar a las personas afectadas
- Formación y capacitación
- Soporte durante la transición
- CelebrarQuick Wins
