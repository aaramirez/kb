# DMAIC: Controlar

## El propósito de Control

La fase Control asegura que las mejoras **se sostienen en el tiempo**. Sin control, las mejoras se degradan y el proceso vuelve a su estado anterior. Es donde se cierra el ciclo de [[05-Vertientes/5b-Lean-Six-Sigma/02-dmaic-metodologia|DMAIC]].

> *"Lo que no se mide, se pierde. Lo que no se controla, se degrada."*

## Control Plans

El Control Plan documenta cómo se mantendrá el proceso mejorado:

| Elemento | Contenido |
|----------|-----------|
| **Especificación** | Límites de aceptación del proceso |
| **Método de monitoreo** | Qué se mide y cómo |
| **Frecuencia** | Cada cuánto se monitorea |
| **Responsable** | Quién revisa y actúa |
| **Acción correctiva** | Qué hacer si hay desviación |
| **Escalamiento** | Cuándo escalar a gerencia |

## Statistical Process Control (SPC)

### Gráficos de control
- **X-bar & R** — para medias y rangos de subgrupos
- **X-bar & S** — para medias y desviación estándar
- **p-chart** — para proporciones de defectos
- **c-chart** — para conteo de defectos por unidad
- **u-chart** — para defectos por unidad variable

### Interpretación
- **En control** — variación dentro de límites esperados (causa común)
- **Fuera de control** — variación por causa especial
- **Patrones** — tendencias, ciclos, puntos consecutivos en un lado

### Límites de control vs especificación
| Tipo | Origen | Uso |
|------|--------|-----|
| Límites de control | Datos del proceso | ¿El proceso es estable? |
| Límites de especificación | Requisitos del cliente | ¿El proceso cumple? |

## Sostenibilidad

### Elementos clave
1. **Documentación** — procedimientos actualizados y accesibles
2. **Formación** — capacitar al personal que ejecuta
3. **Controles visuales** — gráficos en el gemba
4. **Auditorías** — revisiones periódicas
5. **Responsabilidad** — dueños del proceso claros
