# Lean Software Development — Vista General

## ¿Qué es Lean Software Development?

Lean Software Development es una adaptación de los principios de [[05-Vertientes/5a-Manufactura/01-lean-manufactura-vista-general|Lean Manufacturing]] al contexto del desarrollo de software. Propuesta por **Mary y Tom Poppendieck** en 2003, se enfoca en eliminar desperdicios del proceso de desarrollo.

> *"Lean es simplemente eliminar todo lo que no agrega valor para el cliente."* — Mary Poppendieck

## Mary y Tom Poppendieck

Los Poppendieck tomaron los 14 principios de Toyota y los tradujeron a [[05-Vertientes/5d-Lean-Software/02-siete-principios-lean-software|7 principios de Lean Software]]. Su aporte fue demostrar que Lean no es solo para fábricas — también funciona para software.

## Los 7 principios

| # | Principio | Lean Manufacturing original |
|---|-----------|---------------------------|
| 1 | Eliminar desperdicio | Eliminar 8 desperdicios |
| 2 | Amplificar aprendizaje | Kaizen |
| 3 | Decidir lo más tarde posible | Flexibilidad |
| 4 | Entregar lo más rápido posible | One-Piece Flow |
| 5 | Empoderar al equipo | Respeto a las personas |
| 6 | Construir integridad | Jidoka |
| 7 | Ver el todo | Value Stream |

## Conexión con Agile

Lean Software y Agile son **complementarios**, no competidores:

| Aspecto | Agile | Lean Software |
|---------|-------|---------------|
| **Origen** | Manifiesto Agile (2001) | Toyota / Poppendieck (2003) |
| **Enfoque** | Iteraciones y entrega | Eliminar desperdicio |
| **Framework** | Scrum, XP, Kanban | Principios universales |
| **Métricas** | Velocity, burn-down | Lead time, cycle time |
| **Cultura** | Auto-organización | Mejora continua |

- Reducción de tiempo de交付 (delivery)
- Mejora en calidad del código
- Mayor satisfacción del equipo
- Menos tiempo en activities no valoradas
- Mejor alignment con necesidades del cliente
