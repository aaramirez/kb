# MVP (Minimum Viable Product)

## Definición

Un MVP es la versión **más simple** de un producto que permite **iniciar el ciclo Build-Measure-Learn** con el mínimo esfuerzo. No es un producto a medias — es un **experimento** disfrazado de producto.

> *"El MVP no es el producto más pequeño que puedas construir. Es el que más aprendizaje te da."* — Eric Ries

## Propósito del MVP

| Propósito | Descripción |
|-----------|-------------|
| **Validar hipótesis** | ¿Los clientes quieren esto? |
| **Aprender rápido** | ¿Qué features importan? |
| **Reducir riesgo** | ¿Vale la pena seguir invirtiendo? |
| **Generar datos** | Comportamiento real, no opiniones |

## Tipos de MVP

### MVP de VIDEO
- Explicar el producto como si existiera
- Medir interés antes de construir
- Ejemplo: Dropbox's video explicativo

### MVP de LANDING PAGE
- Página que describe el producto
- Medir conversiones (email signup, waitlist)
- Ejemplo: Buffer's landing page

### MVP de CONCIETO (Concierge)
- Servicio manual que parece automatizado
- Aprender flujos antes de construir tech
- Ejemplo: Food on the Table

### MVP de PIEZA MÁGICA
- Automatizar solo una parte del producto
- El resto es manual o no existe
- Ejemplo: Zappos — fotos de zapatos, envío manual

### MVP de OZ (Wizard of Oz)
- Por fuera parece completo
- Por dentro es todo manual
- Validar demanda sin construir infraestructura

## Cómo construir un MVP

1. **Definir la hipótesis** — ¿Qué queremos aprender?
2. **Identificar la métrica** — ¿Cómo sabemos si funciona?
3. **Elegir el tipo de MVP** — ¿Cuál es el más rápido y barato?
4. **Construir** — lo más rápido posible (días/semanas)
5. **Lanzar** — a un grupo pequeño de usuarios
6. **Medir** — recoger datos reales
7. **Aprender** — ¿continuar, pivotar o matar?

## Errores comunes

| Error | Consecuencia |
|-------|-------------|
| Construir demasiado features | Mucho tiempo, poco aprendizaje |
| Esperar a tener un "producto completo" | Meses sin feedback real |
| No tener hipótesis clara | No se puede evaluar si funciona |
| Medir solo opiniones | Feedback no representativo |
| Lanzar a todos a la vez | Riesgo reputacional alto |

## MVP vs Producto Completo

| Aspecto | MVP | Producto completo |
|---------|-----|-------------------|
| **Tiempo** | Días/semanas | Meses/años |
| **Features** | Mínimo esencial | Completo |
| **Riesgo** | Bajo | Alto |
| **Aprendizaje** | Rápido | Lento |
**Costo** | Bajo | Alto |
