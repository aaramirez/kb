# Pivotar o Perseverar

## La decisión más importante

Después de cada ciclo [[05-Vertientes/5c-Lean-Startup/02-construir-medir-aprender|Build-Measure-Learn]], el equipo debe decidir: ¿seguimos en la dirección actual o cambiamos?

> *"Un pivot no es un fracaso. Es una decisión basada en aprendizaje."*

## ¿Cuándo pivotar?

### Señales de que es hora de pivotar
- [[05-Vertientes/5c-Lean-Startup/04-aprendizaje-validado|Aprendizaje validado]] muestra que la hipótesis principal es falsa
- Las métricas clave están estancadas o declining
- Los clientes no valoran lo que construiste
- El mercado es más pequeño de lo esperado
- La tecnología no es viable como se planeó

### Señales de que es hora de perseverar
- Métricas están mejorando lentamente pero consistentemente
- Hay engagement real (no solo registros)
- Los clientes dan feedback constructivo (no rechazo total)
- El mercado es grande y la tendencia es favorable

## Tipos de pivots

| Tipo de Pivot | Descripción | Ejemplo |
|---------------|-------------|---------|
| **Zoom-in** | Una feature se convierte en todo el producto | Instagram de Burbn |
| **Zoom-out** | El producto completo se convierte en una feature | — |
| **Customer segment** | Mismo producto, diferente cliente | Slack de Tiny Speck |
| **Customer need** | Mismo cliente, diferente problema | — |
| **Platform** | Cambiar de aplicación a plataforma (o viceversa) | — |
| **Business architecture** | Cambiar modelo de negocio | B2C → B2B |
| **Value capture** | Cambiar cómo se monetiza | Freemium → Enterprise |
| **Engine of growth** | Cambiar motor de crecimiento | Viral → Paid |
| **Channel** | Cambiar canal de distribución | Online → Retail |
| **Technology** | Misma solución, diferente tecnología | — |

## Framework de decisión

```
¿El aprendizaje validado confirma la hipótesis principal?
  │
  ├── SÍ → Perseverar (iterar, mejorar, escalar)
  │
  └── NO → ¿Hay evidencia de que el mercado existe?
              │
              ├── SÍ → Pivotar (cambiar enfoque, mantener mercado)
              │
              └── NO → Pivotar o abandonar
```

## Cómo pivotar bien

1. **Mantener lo que funciona** — no cambiar todo por cambiar
2. **Preservar aprendizaje** — documentar qué se sabe
3. **Animar al equipo** — un pivot es energía renovada, no fracaso
4. **Comunicar a stakeholders** — transparencia sobre el cambio
5. **Definir nuevas hipótesis** — el siguiente ciclo BML empieza aquí
