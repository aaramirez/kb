# Build-Measure-Learn (Construir-Medir-Aprender)

## El ciclo fundamental

Build-Measure-Learn (BML) es el **motor de retroalimentación** de Lean Startup. Cada ciclo nos acerca a un producto que los clientes realmente quieren.

```
        ┌─── Construir ───┐
        │                  │
   Aprender              Medir
        │                  │
        └─── Medir ────────┘
```

> *"El objetivo no es construir rápido. El objetivo es aprender rápido qué construir."*

## Las 3 fases del ciclo

### 1. Construir (Build)
- Crear un [[05-Vertientes/5c-Lean-Startup/03-mvp-minimo-viable|MVP]] que ponga a prueba la hipótesis
- No es construir el producto final — es construir un **experimento**
- Velocidad: semanas, no meses
- Prioridad: learn, no features

### 2. Medir (Measure)
- Recoger datos del comportamiento real de usuarios
- Usar [[05-Vertientes/5c-Lean-Startup/06-contabilidad-de-innovacion|Innovation Accounting]]
- Evitar "vanity metrics" (métricas vanidosas)
- Priorizar métricas accionables

### 3. Aprender (Learn)
- ¿La hipótesis fue validada o refutada?
- ¿Qué sabemos ahora que no sabíamos antes?
- Decidir: [[05-Vertientes/5c-Lean-Startup/05-pivotar-o-perseverar|pivotar o perseverar]]
- Documentar aprendizaje para toda la organización

## Validated Learning

Cada ciclo BML produce [[05-Vertientes/5c-Lean-Startup/04-aprendizaje-validado|aprendizaje validado]]:

| Tipo de conocimiento | Ejemplo |
|---------------------|---------|
| **Hipótesis de valor** | "Los usuarios pagarán $X por Y" |
| **Hipótesis de problema** | "El problema Z es importante para el cliente" |
| **Hipótesis de solución** | "Nuestra solución resuelve Z mejor que alternativas" |

## Métricas que importan

| Métrica vanidosa | Métrica accionable |
|-----------------|-------------------|
| "Tenemos 1M de usuarios" | "El 30% usa la feature X semanalmente" |
| "Descargas crecen 20%/mes" | "Retention a 30 días es 40%" |
| "Likes en redes sociales" | "Tasa de conversión de trial a pago" |

## Ritmo del ciclo

- **Ideal:** 1 ciclo por semana para startups tempranas
- **MVP funcional:** 2-4 semanas
- **Experimento de diseño:** 1-2 semanas
- **Velocidad > perfección** — un ciclo imperfecto vale más que ningún ciclo

## Errores comunes

- Saltarse la fase "Measure" (construir sin medir)
- Medir solo vanity metrics
- Tomar meses para completar un ciclo
- No documentar aprendizaje
- No tener hipótesis claras antes de construir
