# Last Planner System (LPS)

## ¿Qué es el Last Planner System?

El [[05-Vertientes/5f-Lean-Construction/02-last-planner-system|Last Planner System]] (LPS) es un sistema de planificación y control de proyectos de construcción que busca **aumentar la predictibilidad** del trabajo. Fue desarrollado por Glenn Ballard y Greg Howell como respuesta a la alta tasa de retrasos en la industria.

> *"El Last Planner no es una persona, es un rol: quien toma el compromiso de que el trabajo se realizará."*

## Las cuatro fases de planificación

El LPS funciona como un sistema de filtración que va del plan general al compromiso semanal:

### 1. Planificación de fase (Phase Planning)
- **Horizonte** — todo el proyecto
- **Actividad** — definir hitos y secuencia de construcciones
- **Resultado** — cronograma maestro del proyecto

### 2. Planificación anticipada (Look-Ahead Planning)
- **Horizonte** — 3-6 semanas
- **Actividad** — identificar y eliminar restricciones
- **Resultado** — lista de trabajo preparado para ejecución

### 3. Planificación semanal (Weekly Planning)
- **Horizonte** — 1-2 semanas
- **Actividad** — compromisos de ejecución del equipo
- **Resultado** — plan semanal con compromisos concretos

### 4. Seguimiento de compromisos (Commitment Tracking)
- **Horizonte** — retrospectiva semanal
- **Actividad** — medir cumplimiento y aprender
- **Resultado** — identificar causas de no cumplimiento

## Percent Plan Complete (PPC)

El [[03-Pilares/01-cero-defectos|PPC]] es la métrica fundamental del LPS:

```
PPC = (Compromisos cumplidos / Total de compromisos) × 100%
```

- **Objetivo** — mantener PPC por encima del 80%
- **Análisis** — cuando no se cumple, investigar causas raíz
- **Tendencia** — mejorar PPC gradualmente con cada ciclo

### Causas comunes de bajo PPC
- Restricciones no eliminadas a tiempo
- Dependencias no identificadas
- Falta de recursos
- Cambios en el diseño
- Condiciones climáticas adversas

## Beneficios para proyectos

| Beneficio | Descripción |
|-----------|-------------|
| **Mayor predictibilidad** | Menos sorpresas en la ejecución |
| **Mejor coordinación** | Todos los subcontratistas alineados |
| **Reducción de desperdicios** | Menos espera y retrabajo |
| **Cultura de compromiso** | Responsabilidad compartida |
| **Aprendizaje organizacional** | Mejora continua del sistema |

## Implementación paso a paso

1. **Capacitar al equipo** — enseñar los principios del LPS
2. **Establecer la estructura** — definir roles y responsabilidades
3. **Iniciar con un piloto** — probar en una fase del proyecto
4. **Medir y ajustar** — monitorear PPC y causas de falla
5. **Expandir gradualmente** — aplicar a todo el proyecto
