# Scaling Lean-Agile (Escalamiento)

## Desafíos del escalamiento

Cuando una organización crece, los principios Lean y Agile enfrentan nuevos retos:

- **Coordinación** entre múltiples equipos
- **Dependencias** técnicas y de negocio
- **Consistencia** en prácticas y calidad
- **Visibilidad** a nivel de portfolio
- **Velocidad** sin perder calidad

## Modelos de escalamiento

### SAFe (Scaled Agile Framework)
El más adoptado a nivel mundial. Combina Lean, Agile y DevOps en un marco integral con niveles: Team, Program, Large Solution, Portfolio.

### LeSS (Large-Scale Scrum)
Minimalismo extremo. Agrega lo mínimo necesario para escalar Scrum. Menos roles, menos artefactos, menos ceremonias. Coopy by Decomposition.

### Nexus
Creado por Ken Schwaber (co-creador de Scrum). Integra 3-9 equipos Scrum con un Nexus Integration Team.

### Modelo Spotify
No es un framework formal, sino un modelo organizacional basado en:
- **Squads** — equipos autónomos
- **Tribes** — agrupaciones de squads
- **Chapters** — comunidades de práctica
- **Guilds** — redes transversales

## Portfolio Lean-Agile

El portfolio management Lean-Agile se enfoca en:

```
Estrategia
    ↓
Value Stream Map (a nivel portfolio)
    ↓
Iniciativas priorizadas por costo de retraso
    ↓
Ejecución via ARTs o equipos
    ↓
Medición de resultados
```

## Diseño organizacional

Lean-Agile promueve estructuras basadas en **flujos de valor**, no en funciones:

| Modelo tradicional | Modelo Lean-Agile |
|-------------------|-------------------|
| Departamentos por función | Equipos por value stream |
| Gestión por capas | Liderazgo servant |
| Presupuesto anual por depto | Fondos por iniciativa |
| Planificación top-down | Planificación participativa |

## Cuándo escalar

Escalar solo cuando:

- Un equipo solo no puede entregar valor suficiente
- Hay dependencias reales entre equipos
- La coordinación manual consume demasiado tiempo
- Se requiere alineación estratégica a nivel de portfolio
