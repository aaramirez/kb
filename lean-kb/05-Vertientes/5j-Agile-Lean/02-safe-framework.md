# SAFe Framework (Scaled Agile Framework)

## ¿Qué es SAFe?

**SAFe** (Scaled Agile Framework) es un marco de trabajo para implementar Agile a escala en organizaciones grandes. Fue creado por Dean Leffingwell y combina principios Lean, pensamiento de sistemas y desarrollo ágil en un modelo integral.

## Principios Lean-Agile en SAFe

SAFe se basa en **10 principios** derivados de Lean y Agile:

1. **Adoptar una mentalidad económica** — tomar decisiones considerando el costo del retraso
2. **Pensar en sistemas** — optimizar el todo, no las partes
3. **Suponer variabilidad, conservar opciones** — mantener alternativas abiertas
4. **Construir rápidamente ciclos de aprendizaje** — cortos e integrados
5. **Basar las milestones en la evaluación de sistemas objetivos** — medir resultados reales
6. **Hacer visible el trabajo del sistema** — transparencia total
7. **Aplicar cadencia, sincronizar con la跨领域** — ritmo predecible
8. **Desbloquear la motivación intrínseca de los trabajadores** — autonomía y propósito
9. **Descentralizar la toma de decisiones** — descentralizar la autoridad

## Agile Release Train (ART)

El ART es la unidad fundamental de SAFe. Es un equipo virtual de 50-125 personas que planifica, entrega y mejora de forma sincronizada.

```
┌─────────────────────────────────────────┐
│           Agile Release Train           │
│                                         │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │Team 1│ │Team 2│ │Team 3│ │Team N│  │
│  └──────┘ └──────┘ └──────┘ └──────┘  │
│                                         │
│  ┌──────────────────────────────────┐   │
│  │    Inspect & Adapt Workshop      │   │
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

Cada ART opera en un **Program Increment** (PI) de 8-12 semanas, con eventos sincronizados.

## Gestión de Portfolio

SAFe Lean Portfolio Management (LPM) conecta la estrategia con la ejecución:

- **Estrategia e inversión**: definir qué construir
- **Gestión de flujo de valor**: optimizar el delivery
- **Gobernanza**: decisiones descentralizadas con alignment

## Cuándo usar SAFe

SAFe es apropiado cuando:

- Hay múltiples equipos que deben sincronizarse
- La organización necesita visibilidad a nivel de portfolio
- Se requiere gobierno y compliance junto con agilidad
- El sector es regulado pero se busca innovación

## Limitaciones

- Puede ser pesado para equipos pequeños
- Requiere coaching significativo
- El riesgo de burocracia es real si no se aplica con cuidado
