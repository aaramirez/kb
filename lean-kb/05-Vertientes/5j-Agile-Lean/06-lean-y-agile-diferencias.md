# Lean vs Agile - Diferencias

## Orígenes distintos

| Aspecto | Lean | Agile |
|---------|------|-------|
| **Origen** | Toyota, Japón, años 50 | Manifiesto Ágil, 2001, USA |
| **Creador(es)** | Taiichi Ohno, Shigeo Shingo | 17 signatarios del Manifiesto |
| **Industria original** | Manufactura | Software |
| **Filosofía base** | Eliminar desperdicio | Entregar valor rápido |

## Enfoque principal

**Lean** se enfoca en:
- Optimizar todo el **flujo de valor** (end-to-end)
- Eliminar los **8 desperdicios** (muda)
- Crear sistemas **pull** en toda la organización
- Mejora continua como **cultura**

**Agile** se enfoca en:
- Entregar **software funcionando** rápidamente
- **Iteraciones cortas** con feedback
- **Equipos auto-organizados**
- **Colaboración** con el cliente

## Escala de impacto

```
Lean:        Organización completa → Proveedores → Clientes
Agile:       Equipo → Proyecto → Producto
```

Lean tiene un alcance sistémico; Agile típicamente opera a nivel de equipo o producto.

## Métricas

| Lean | Agile |
|------|-------|
| Lead Time, Cycle Time | Velocity, Burn-down |
| Overall Equipment Effectiveness | Sprint Goal achievement |
| Cost of Delay | Story Points delivered |
| First Pass Yield | Code Coverage |

## Cuándo usar cada uno

### Usar Lean cuando:
- Se necesita transformar toda la organización
- El desperdicio es el problema principal
- Se busca excelencia operativa
- Hay procesos de manufactura o servicios

### Usar Agile cuando:
- El equipo de desarrollo necesita trabajar mejor
- Se requiere velocidad de entrega
- El mercado cambia rápidamente
- El feedback del cliente es crítico

### Usar ambos cuando:
- Se busca agilidad **y** eficiencia
- La organización es compleja
- Se necesita escalar ágilmente
- Se quiere una transformación completa

## Complementariedad

Lean y Agile no son excluyentes. De hecho, lo más efectivo es combinarlos:

- Lean da la **visión sistémica**
- Agile da la **velocidad de ejecución**
- Juntos crean organizaciones **rápidas y eficientes**
