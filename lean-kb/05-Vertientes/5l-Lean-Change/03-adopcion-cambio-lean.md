# Adopción del Cambio con Lean

## Enfoque Lean para la adopción

Lean no impone cambios; los **experimenta**. La adopción se logra mediante ciclos pequeños de prueba, aprendizaje y ajuste.

## Experimentos y pilotos

### Diseño de experimentos de cambio

1. **Hipótesis**: "Si implementamos X, veremos Y en Z semanas"
2. **Métrica**: ¿cómo sabremos si funcionó?
3. **Duración**: corta (1-4 semanas)
4. **Alcance**: un equipo o un proceso
5. **Decisión**: ¿escalar, ajustar o abandonar?

### Ejemplo práctico

```
Hipótesis: "Si implementamos daily standups de 15 minutos,
           reduciremos el cycle time en 20%"

Métrica: Cycle time promedio por tarea
Duración: 3 sprints
Alcance: Equipo de desarrollo
Resultado: 15% de reducción → ajustar y continuar
```

## Feedback loops en el cambio

El cambio necesita **ciclos de retroalimentación** constantes:

```
Experimento → Feedback → Ajuste → Nuevo experimento
     ↑                                    ↓
     └────────────────────────────────────┘
```

### Fuentes de feedback

- **Datos cuantitativos**: métricas de adopción
- **Observación directa**: Gemba walk del cambio
- **Conversaciones**: retros del cambio
- **Señales**: rotación, ausentismo, calidad

## Midiendo la adopción del cambio

### Métricas de adopción

| Tipo | Ejemplo |
|------|---------|
| **Uso** | % de equipo usando la nueva práctica |
| **Frecuencia** | Cuántas veces se usa por semana |
| **Calidad** | ¿Se usa correctamente? |
| **Resultados** | ¿Mejoran las métricas objetivo? |
| **Satisfacción** | ¿El equipo valora el cambio? |

### Evitar métricas de vanidad

No medir solo "se implementó", sino "generó valor". Un cambio implementado que nadie usa es desperdicio.

## Aprendizaje como resultado

En Lean Change, el **aprendizaje** es tan valioso como el éxito:

- Si el experimento funcionó → escalar
- Si no funcionó → ¿qué aprendimos?
- Si parcialmente → ¿qué ajustar?
