# Lean Change - Vista General

## ¿Qué es Lean Change Management?

**Lean Change Management** es un enfoque que aplica los principios Lean a la gestión del cambio organizacional. En lugar de planificar cambios extensos y luego implementarlos, se trata el cambio como una serie de **experimentos pequeños** con feedback continuo.

## Cambio como proceso a optimizar

En Lean, todo proceso tiene desperdicio. El cambio organizacional no es la excepción:

```
Cambio tradicional:        Cambio Lean:
Planificar (meses)         Experimentar (días)
Implementar (meses)        Medir resultado
Evaluar (años)             Ajustar
                           Repetir
```

## Principios clave

### 1. Cambio incremental
No grandes revoluciones, sino mejoras pequeñas y frecuentes. Cada experimento enseña algo.

### 2. Experimentación
No "sabemos qué funciona", sino "vamos a probar qué funciona". Hipótesis → Experimento → Datos → Decisión.

### 3. Feedback loops
Cortos y frecuentes. No esperar al final de un año para evaluar si el cambio funcionó.

### 4. Respeto por las personas
El cambio se hace **con** las personas, no **a** las personas. Su conocimiento y resistencia son datos valiosos.

### 5. Visualización
Hacer visible el progreso del cambio, los bloqueadores, y los aprendizajes.

## Framework ODC

Lean Change Management usa el ciclo **Observe-Direct-Change**:

1. **Observe**: entender el estado actual del cambio
2. **Direct**: definir la dirección deseada
3. **Change**: ejecutar experimentos para avanzar

## Diferencia con change management tradicional

| Tradicional | Lean Change |
|-------------|-------------|
| Top-down | Participativo |
| Plan de 18 meses | Experimentos de 2 semanas |
| Resistencia = problema | Resistencia = información |
| Éxito = adopción total | Éxito = aprendizaje |
| Communication plan | Conversación continua |
