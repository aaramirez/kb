# Value (Valor)

## Definicion

En el contexto de Lean, el **[[06-valor|Valor]]** se define exclusivamente desde la perspectiva del cliente. Es cualquier actividad, caracteristica o servicio por el cual el cliente esta dispuesto a pagar.

> **Valor** = Lo que el cliente necesita + Lo que el cliente esta dispuesto a pagar

## Perspectiva del Cliente

El valor no lo define la empresa, sino el cliente. Esto significa:

- Lo que la empresa cree que es importante puede no serlo para el cliente
- Las especificaciones tecnicas no son sinonimo de [[06-valor|valor]]
- El verdadero valor esta en resolver necesidades reales
- Cada cliente puede tener una definicion ligeramente diferente

## Valor vs Desperdicio (Muda)

La forma mas simple de entender el [[06-valor|valor]] es distinguirlo del desperdicio:

### Actividades de Valor (Value-Added)

- Transforman el producto o servicio de alguna forma
- El cliente esta dispuesto a pagar por ellas
- Se hacen correctamente a la primera

### Actividades Sin Valor (Non-Value-Added)

- No transforman el producto
- El cliente no pagaria por ellas
- Son puro desperdicio (Muda)

### Actividades Necesarias Sin Valor

- No agregan [[06-valor|valor]] directamente
- Son necesarias por razones regulatorias o de seguridad
- Deben minimizarse

## Los 7 Desperdicios que Reducen el Valor

Los 7 desperdicios identificados por [[13-pensadores-lean-clave|Taiichi Ohno]] son:

1. **Sobreproduccion** - Producir mas de lo necesario
2. **Espera** - Tiempo muerto entre pasos
3. **Transporte** - Mover materials innecesariamente
4. **Procesamiento excesivo** - Hacer mas de lo requerido
5. **Inventario** - Productos sin vender
6. **Movimiento** - Movimientos innecesarios de personas
7. **Defectos** - Errores que requieren retrabajo

Cada uno de estos reduce el [[06-valor|valor]] total percibido por el cliente.

## Como Identificar el Valor

Preguntas clave para identificar el [[06-valor|valor]]:

- Que necesita realmente el cliente?
- Que problema resuelve este producto/servicio?
- Que caracteristicas son esenciales?
- Que estaria dispuesto a pagar extra?
- Que consideraria desperdicio?

## Ejemplos de Valor en Diferentes Contextos

### Manufactura

- Ensamblar componentes en un producto terminado
- Pintar una pieza segun especificacion
- Empaquetar para proteger durante el transporte

### Servicios

- Resolver un problema del cliente
- Procesar una solicitud correctamente
- Entregar informacion relevante y util

### Software

- Funcionalidades que el usuario usa
- Rendimiento que mejora la experiencia
- Seguridad que protege los datos

## El Valor no es Estatico

El [[06-valor|valor]] cambia con el tiempo:

- Las necesidades del cliente evolucionan
- La tecnologia crea nuevas expectativas
- La competencia redefine estandares
- El contexto social influye en las percepciones

Por eso el [[10-perfeccion|Kaizen continuo]] es esencial.

## Valor y el Primer Principio

Como primer principio de los [[05-cinco-principios-lean|5 principios de Lean]], el [[06-valor|valor]] establece la direccion para todo lo demas:

- Sin entender el [[06-valor|valor]], no se puede mapear la [[07-cadena-de-valor|cadena de valor]]
- Sin la [[07-cadena-de-valor|cadena de valor]], no se puede crear [[08-flujo|flujo]]
- Sin [[08-flujo|flujo]], no se puede implementar un [[09-sistema-de-tiron|sistema de tiron]]
- Sin un [[09-sistema-de-tiron|sistema de tiron]], no se puede buscar la [[10-perfeccion|perfeccion]]

## Errores Comunes

- Definir valor desde adentro (no desde el cliente)
- Confundir actividad con [[06-valor|valor]]
- Asumir que mas caracteristicas = mas [[06-valor|valor]]
- Ignorar la variabilidad en las necesidades del cliente
- No actualizar la definicion de [[06-valor|valor]] con el tiempo
