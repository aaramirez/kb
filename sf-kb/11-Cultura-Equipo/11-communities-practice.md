## Definición

Una Community of Practice (CoP) es un grupo de personas que comparten una preocupación o pasión por un topic y profundizan su knowledge y expertise interacción regularmente (Wenger & Wenger, 1998).

### Elementos fundamentales

| Elemento | Description |
|----------|-------------|
| **Domain** | El topic compartido (ej: frontend, ML, security) |
| **Community** | Las personas que participan activamente |
| **Practice** | El cuerpo compartido de knowledge, tools, approaches |

## CoPs en Software Engineering

| Community | Domain | Ejemplo de activities |
|-----------|--------|----------------------|
| **Frontend Guild** | React, CSS, UX | Component library, design system |
| **Backend Guild** | APIs, architecture | API guidelines, service patterns |
| **DevOps Community** | CI/CD, infra | Pipeline standards, tooling |
| **Security Guild** | Application security | Secure coding, threat modeling |
| **Data Engineering** | Pipelines, analytics | Data standards, quality practices |
| **Testing Guild** | QA, automation | Test strategies, tool selection |

## CoP vs Chapter vs Guild

| Unidad | Scope | Membership | Purpose |
|--------|-------|------------|---------|
| **Chapter** | Within a tribe | Same skill set, same org unit | Best practices, people management |
| **Guild** | Cross-tribe | Interest-based, voluntary | Knowledge sharing, standards |
| **CoP** | Organization-wide | Voluntary, passion-driven | Deep expertise, innovation |

## How to Start a CoP

### Phase 1: Seed (Month 1)

| Activity | Owner | Timeline |
|----------|-------|----------|
| Identify passionate advocates | Founder | Week 1 |
| Define domain and value proposition | Founder | Week 1-2 |
| Recruit initial members (5-10) | Founder | Week 2-3 |
| Choose communication channels | Founder | Week 3 |
| Plan first event/meetup | Founder | Week 4 |

### Phase 2: Grow (Months 2-6)

| Activity | Cadence |
|----------|---------|
| Regular meetings | Quincenal o mensual |
| Knowledge sharing sessions | Cada meeting |
| Internal blog/documentation | Semanal |
| Collaboration on shared artifacts | Ongoing |
| External speaker invitations | Trimestral |

### Phase 3: Sustain (Month 6+)

| Activity | Purpose |
|----------|---------|
| Rotating leadership | Prevent burnout, distribute ownership |
| Onboarding for new members | Welcome path |
| Annual planning | Strategic direction |
| Metrics review | Measure impact |
| Cross-CoP collaboration | Prevent silos between CoPs |

## Governance Models

| Model | Structure | Best for |
|-------|-----------|----------|
| **Organic** | No formal structure, self-organizing | Small orgs, passionate groups |
| **Rotating lead** | Leadership rotates quarterly | Medium orgs, preventing single point |
| **Steering committee** | 3-5 leads make decisions | Large orgs, resource allocation |
| **Sponsored** | Leadership sponsors + community leads | Enterprise, needs budget/headcount |

### Meeting Format Template

| Phase | Duration | Activity |
|-------|----------|----------|
| **Check-in** | 5 min | Quick round, how everyone is doing |
| **Main topic** | 30-40 min | Tech talk, demo, or discussion |
| **Open floor** | 15 min | Questions, sharing, announcements |
| **Action items** | 5 min | Next steps, follow-ups |

## Success Metrics

| Metric | Target | How to measure |
|--------|--------|---------------|
| **Active members** | > 20 per CoP | Attendance tracking |
| **Meeting attendance** | > 60% average | Sign-in / Zoom data |
| **Knowledge artifacts** | > 1/month | Published docs, tools |
| **Cross-team adoption** | Growing usage | Tool/library adoption metrics |
| **Member satisfaction** | > 4.0/5 | Quarterly survey |
| **New member onboarding** | < 2 weeks to first contribution | Time tracking |

## Anti-Patterns

| Anti-Patrón | Consecuencia |
|-------------|-------------|
| **Mandatory attendance** | Kills intrinsic motivation |
| **No clear domain** | Confusion, low engagement |
| **Founder dependency** | Single point of failure |
| **Talk-only, no artifacts** | Knowledge doesn't persist |
| **Ivory tower** | Disconnected from real work |
| **Too many CoPs** | Fragmentation, fatigue |
