## ROI Calculation

### Formula

```
AI ROI = (Gains - Costs) / Costs Г— 100

Gains = (Time saved Г— Developer cost) + (Reduced defects Г— Fix cost) + (Faster delivery Г— Business value)

Costs = Tool licenses + Training + Setup + Maintenance
```

### Ejemplo de cГЎlculo

```
Team: 10 developers
Average salary: $120K/year ($60/hour)

Time Savings:
в”ңв”Җв”Җ Code completion: 30 min/day Г— 10 devs Г— 250 days = 1,250 hours
в”ңв”Җв”Җ Code review: 1h/PR Г— 50 PRs/month Г— 12 months = 600 hours
в”ңв”Җв”Җ Documentation: 2h/week Г— 10 devs Г— 52 weeks = 1,040 hours
в””в”Җв”Җ Total time saved: 2,890 hours Г— $60 = $173,400

Quality Savings:
в”ңв”Җв”Җ Reduced bugs: 20 bugs/month Г— $500 avg fix Г— 12 = $120,000
в””в”Җв”Җ Security fixes: 5 critical Г— $5,000 avg Г— 12 = $300,000

Business Value:
в”ңв”Җв”Җ Faster delivery: 2 months earlier Г— $50K/month = $100,000
в””в”Җв”Җ Total gains: $723,400

Costs:
в”ңв”Җв”Җ Copilot licenses: $39/user Г— 10 Г— 12 = $4,680
в”ңв”Җв”Җ Training: $5,000 (one-time)
в”ңв”Җв”Җ Setup: $3,000 (one-time)
в””в”Җв”Җ Total costs: $12,680

ROI = ($723,400 - $12,680) / $12,680 Г— 100 = 5,604%
```

## Best Practices for Measurement

1. **Baseline first**: Measure before AI adoption
2. **Control group**: Compare with non-AI team (if possible)
3. **Leading + lagging indicators**: Both velocity and quality
4. **Regular cadence**: Monthly metrics review
5. **Qualitative feedback**: Surveys, retrospectives
6. **Long-term view**: 6-12 months for meaningful data
7. **Context matters**: Same metrics may mean different things

## Anti-patterns de MediciГіn

| Anti-pattern | Why it fails | Better approach |
|--------------|-------------|-----------------|
| Only measuring velocity | Can increase bugs | Include quality metrics |
| Before/after without control | Confounding factors | Use control group |
| Monthly snapshot only | No trend data | Continuous tracking |
| Self-reported productivity | Bias | Use tool metrics |
| Ignoring qualitative | Numbers don't tell all | Include surveys |
