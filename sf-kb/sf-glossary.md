# Glosario de Software Factory

Glosario maestro con los términos, siglas y conceptos clave utilizados en esta knowledge base.

## A

- **ADR (Architecture Decision Record)** — Documento que registra decisiones arquitectónicas importantes. Ver [[10-Gestion-Proyectos/09-adr-decisiones|ADR]]
- **Agile** — Familia de metodologías basadas en el Manifiesto Ágil. Ver [[01-Fundamentos/06-manifiesto-agile|Manifiesto Ágil]]
- **AIOps** — Inteligencia artificial aplicada a operaciones de TI. Ver [[07-IA-Software/23-ai-devops-integration|AIOps]]
- **API (Application Programming Interface)** — Interfaz de programación de aplicaciones
- **Architectural Fitness Functions** — Funciones que validan restricciones arquitectónicas
- **Auto-scaling** — Escalado automático de recursos

## B

- **Backlog** — Lista priorizada de trabajo por hacer
- **BDD (Behavior-Driven Development)** — Desarrollo orientado a comportamiento. Ver [[04-Practicas/04-behavior-driven-development|BDD]]
- **Branch** — Línea de desarrollo independiente en un repositorio

## C

- **Chaos Engineering** — Práctica de inyectar fallos para mejorar resiliencia. Ver [[04-Practicas/14-chaos-engineering|Chaos Engineering]]
- **CI/CD (Continuous Integration/Continuous Delivery)** — Integración y despliegue continuo. Ver [[03-Procesos/05-ci-cd-pipeline|CI/CD]]
- **Clean Code** — Código que es fácil de leer y mantener. Ver [[04-Practicas/02-clean-code|Clean Code]]
- **Code Review** — Revisión de código por pares antes de merge. Ver [[03-Procesos/08-code-review-proceso|Code Review]]
- **Code Coverage** — Porcentaje de código cubierto por tests. Ver [[06-Metricas/05-code-coverage|Code Coverage]]
- **Cognitive Load** — Carga cognitiva que un sistema o proceso impone al equipo
- **Compliance** — Cumplimiento de regulaciones y estándares. Ver [[08-Calidad-Seguridad/08-compliance-regulaciones|Compliance]]
- **CrewAI** — Framework para crear agentes multi-agente. Ver [[07-IA-Software/18-multi-agent-systems|Multi-Agent Systems]]
- **Cycle Time** — Tiempo desde que se empieza a trabajar hasta que se entrega. Ver [[06-Metricas/03-cycle-time|Cycle Time]]

## D

- **Definition of Done (DoD)** — Criterios que deben cumplirse para considerar una tarea completa. Ver [[04-Practicas/11-definition-of-done|DoD]]
- **DevOps** — Cultura y prácticas que unen desarrollo y operaciones. Ver [[09-DevOps-Infra/01-devops-ciclo|DevOps]]
- **DevSecOps** — Integración de seguridad en DevOps. Ver [[08-Calidad-Seguridad/07-devsecops|DevSecOps]]
- **DORA Metrics** — 4 métricas clave de desempeño de software. Ver [[06-Metricas/01-dora-metrics|DORA]]

## E

- **Error Budget** — Tolerancia aceptable de fallos en un período. Ver [[10-Gestion-Proyectos/09-sla-slo-error-budgets|Error Budgets]]
- **Event-Driven Architecture** — Arquitectura basada en eventos

## F

- **Feature Flag** — Mecanismo para habilitar/deshabilitar funcionalidades sin redeploy. Ver [[04-Practicas/13-feature-flags|Feature Flags]]
- **Flujo Continuo** — Trabajo que avanza sin interrupciones. Ver [[03-Procesos/14-flujo-tirón-pull|Flujo de Tirón]]

## G

- **GitOps** — Usar Git como fuente única de verdad para infraestructura. Ver [[09-DevOps-Infra/07-gitops|GitOps]]
- **GraphQL** — Lenguaje de consulta para APIs

## H

- **Horizontal Scaling** — Agregar más instancias en lugar de más recursos a una instancia

## I

- **IaC (Infrastructure as Code)** — Infraestructura gestionada como código. Ver [[09-DevOps-Infra/03-infrastructure-as-code|IaC]]
- **IDE (Integrated Development Environment)** — Entorno de desarrollo integrado. Ver [[05-Herramientas/01-ides-editores|IDEs]]
- **Incident Response** — Proceso de respuesta a incidentes. Ver [[09-DevOps-Infra/09-incident-response|Incident Response]]
- **Internal Developer Platform (IDP)** — Plataforma interna para desarrolladores. Ver [[02-Estructura/14-plataforma-interna|IDP]]

## J

- **Jidoka** — Automatización con toque humano (del Lean)

## K

- **Kanban** — Sistema visual de gestión del trabajo. Ver [[03-Procesos/02-kanban-software|Kanban]]
- **Key Results** — Resultados medibles en OKRs

## L

- **LangChain** — Framework para construir aplicaciones con LLMs. Ver [[07-IA-Software/21-ai-tools-ecosystem|LangChain]]
- **Lead Time** — Tiempo desde que se solicita hasta que se entrega. Ver [[06-Metricas/03-cycle-time|Lead Time]]
- **LLM (Large Language Model)** — Modelo de lenguaje de gran escala
- **Load Testing** — Pruebas de carga. Ver [[08-Calidad-Seguridad/04-performance-testing|Load Testing]]

## M

- **MCP (Model Context Protocol)** — Protocolo para conectar LLMs con herramientas externas. Ver [[07-IA-Software/19-mcp-protocol|MCP]]
- **Microservices** — Arquitectura de servicios pequeños e independientes. Ver [[01-Fundamentos/12-monolito-microservicios|Microservicios]]
- **Mob Programming** — Programación en grupo. Ver [[04-Practicas/05-pair-programming|Mob Programming]]
- **MTTR (Mean Time to Recovery)** — Tiempo promedio de recuperación. Ver [[06-Metricas/13-mean-time-recovery|MTTR]]

## N

- **NPS (Net Promoter Score)** — Métrica de satisfacción del cliente. Ver [[06-Metricas/08-customer-satisfaction|NPS]]

## O

- **Observabilidad** — Capacidad de entender el estado interno de un sistema. Ver [[04-Practicas/12-observabilidad-practicas|Observabilidad]]
- **OKR (Objectives and Key Results)** — Marco de objetivos y resultados clave. Ver [[10-Gestion-Proyectos/10-okr-software|OKRs]]
- **OWASP (Open Worldwide Application Security Project)** — Proyecto de seguridad de aplicaciones. Ver [[08-Calidad-Seguridad/06-owasp-top10|OWASP]]

## P

- **Pair Programming** — Programación en parejas. Ver [[04-Practicas/05-pair-programming|Pair Programming]]
- **PDCA (Plan-Do-Check-Act)** — Ciclo de mejora continua
- **Production Readiness Review** — Evaluación antes de producción
- **Prompt Engineering** — Arte de comunicarse efectivamente con LLMs. Ver [[07-IA-Software/07-prompt-engineering-dev|Prompt Engineering]]

## Q

- **Quality Gates** — Puntos de verificación de calidad en CI/CD. Ver [[08-Calidad-Seguridad/11-quality-gates|Quality Gates]]

## R

- **RAG (Retrieval-Augmented Generation)** — Generación aumentada con recuperación de información
- **React Agent** — Patrón de agente que razona y actúa. Ver [[07-IA-Software/16-arquitectura-agentes|Arquitecturas de Agentes]]
- **Runbook** — Documento con procedimientos operativos

## S

- **SAST (Static Application Security Testing)** — Testing de seguridad estático
- **SCA (Software Composition Analysis)** — Análisis de composición de software
- **Scrum** — Framework ágil de gestión de proyectos. Ver [[03-Procesos/01-scrum|Scrum]]
- **Security Testing** — Pruebas de seguridad. Ver [[08-Calidad-Seguridad/05-security-testing|Security Testing]]
- **SLO (Service Level Objective)** — Objetivo de nivel de servicio. Ver [[10-Gestion-Proyectos/09-sla-slo-error-budgets|SLOs]]
- **SLA (Service Level Agreement)** — Acuerdo de nivel de servicio. Ver [[10-Gestion-Proyectos/09-sla-slo-error-budgets|SLAs]]
- **Smoke Testing** — Pruebas básicas de funcionalidad
- **SRE (Site Reliability Engineering)** — Ingeniería de confiabilidad del sitio. Ver [[09-DevOps-Infra/08-site-reliability|SRE]]
- **Squad** — Equipo auto-organizado (Modelo Spotify). Ver [[02-Estructura/07-squad-tribe-chapter|Squads]]

## T

- **TDD (Test-Driven Development)** — Desarrollo orientado a pruebas. Ver [[04-Practicas/03-test-driven-development|TDD]]
- **Technical Debt** — Deuda técnica. Ver [[01-Fundamentos/09-technical-debt|Technical Debt]]
- **Trunk-Based Development** — Desarrollo basado en trunk. Ver [[03-Procesos/06-trunk-based-development|Trunk-Based]]

## V

- **Value Stream Mapping** — Mapeo de cadena de valor. Ver [[03-Procesos/13-value-stream-mapping|VSM]]
- **Velocity** — Velocidad del equipo en story points por sprint. Ver [[06-Metricas/02-velocity-throughput|Velocity]]

## W

- **Zero Trust** — Modelo de seguridad sin confianza implícita. Ver [[08-Calidad-Seguridad/09-zero-trust|Zero Trust]]
