# Comparación de Modelos de Cambio

## Resumen ejecutivo

Este documento compara cuatro modelos de gestión del cambio, cada uno con un enfoque diferente pero complementario para manejar la transformación organizacional. No existe un modelo "correcto" — la elección depende del contexto, la cultura organizacional y la naturaleza del cambio.

## Enfoque y filosofía

### Kotter — Transformación dirigida desde la cima

| Aspecto | Descripción |
|---------|-------------|
| **Filosofía** | El cambio mayor requiere liderazgo fuerte de la dirección ejecutiva |
| **Metáfora** | Un viaje con etapas definidas — "La Gran Marcha" |
| **Suposición** | Las personas necesitan ver una crisis o amenaza para cambiar |
| **Mecanismo** | Urgencia → Coalición → Visión → Comunicación → Acción → Victorias → Consolidación → Cultura |
| **Tiempo de implementación** | De 3 a 10 años para cambio cultural profundo |

> "El cambio mayor no ocurre fácilmente por una larga lista de razones."
> — John P. Kotter

**Fortalezas:**
- Modelo bien investigado con décadas de evidencia
- Enfoque completo desde la urgencia hasta la cultura
- Reconoce la importancia del liderazgo ejecutivo
- Proporciona un marco claro y secuencial

**Limitaciones:**
- Percibido como rígido o lineal
- Depende fuertemente de liderazgo ejecutivo
- No aborda la dimensión emocional individual
- Puede ser inaceptable en entornos de alta complejidad

### Lean Change — Experimentación iterativa

| Aspecto | Descripción |
|---------|-------------|
| **Filosofía** | Co-crear el cambio con las personas afectadas |
| **Metáfora** | Un jardín que se cultiva — experimentar, aprender, ajustar |
| **Suposición** | Las personas tienen miedo de experimentar porque los planes tradicionales les dieron permiso para no pensar |
| **Mecanismo** | Hallazgos → Plan de Experiencia → Ejecución → (iterar) |
| **Tiempo de implementación** | Iterativo — ciclos cortos con aprendizaje continuo |

> "La mejor manera de gestionar la resistencia es no preocuparse demasiado por la resistencia."
> — Jason Little

**Fortalezas:**
- Acepta la complejidad y no linealidad del cambio
- Involucra a las personas desde el inicio
- Reduce la resistencia a través de la co-creación
- Compatible con metodologías ágiles
- Permite aprendizaje continuo

**Limitaciones:**
- Requiere una cultura organizacional abierta a la experimentación
- Puede percibirse como careciendo de estructura clara
- No provee un marco completo de liderazgo como Kotter
- Depende de la voluntad de la dirección de ceder control

### Change Agility — Capacidad organizacional

| Aspecto | Descripción |
|---------|-------------|
| **Filosofía** | La agilidad es una capacidad que se puede desarrollar |
| **Metáfora** | Un atleta que entren músculos diferentes |
| **Suposición** | Las organizaciones necesitan desarrollar habilidades específicas para manejar el cambio |
| **Mecanismo** | Visión + Estrategia + Táctica + Aprendizaje continuo |
| **Tiempo de implementación** | Continuo — desarrollo de capacidades a largo plazo |

> "Una organización tiene Change Agility cuando puede identificar y adoptar rápidamente los cambios necesarios."
> — Adkins & Spayd

**Fortalezas:**
- Enfoque en desarrollo de capacidades sostenibles
- Los 12 principios de gestión guían la práctica diaria
- Reconoce el rol crucial del gerente de nivel intermedio
- Integra aprendizaje continuo como principio fundamental

**Limitaciones:**
- Modelo relativamente nuevo (2015)
- Enfoque principalmente en coaching — puede no abordar todas las dimensiones
- Requiere inversión significativa en desarrollo de personas
- Menos evidencia empírica que Kotter

### ADKAR — Cambio individual paso a paso

| Aspecto | Descripción |
|---------|-------------|
| **Filosofía** | El cambio organizacional depende del cambio individual |
| **Metáfora** | Un viaje personal de transformación |
| **Suposición** | Si alguna de las 5 etapas falla, el cambio no será exitoso |
| **Mecanismo** | Awareness → Desire → Knowledge → Ability → Reinforcement |
| **Tiempo de implementación** | Variable — depende del individuo y la complejidad del cambio |

> "Cada fase representa un resultado que necesitamos para lograr para que el cambio tenga éxito."
> — Jeff Hiatt

**Fortalezas:**
- Modelo claro y fácil de comunicar
- Enfoque en el individuo — reconoce que el cambio es personal
- Útil para diagnosticar por qué un cambio no está funcionando
- Ampliamente utilizado en organizaciones de todo el mundo

**Limitaciones:**
- Asume linealidad (en la práctica, las etapas se superponen)
- No aborda completamente la dinámica de sistemas y equipos
- Puede ser demasiado simplista para transformaciones complejas
- No maneja la complejidad organizacional directamente

## Cuándo usar cada modelo

### Escenario 1: Transformación a gran escala en una organización tradicional

**Modelo recomendado: [[01-modelo-kotter|Kotter]]**

| Situación | Justificación |
|-----------|--------------|
| La organización tiene una cultura de mando y control | Kotter reconoce la necesidad de liderazgo ejecutivo |
| Hay resistencia significativa al cambio | Los 8 pasos abordan la resistencia sistemáticamente |
| Se necesita un marco claro y estructurado | Kotter proporciona una hoja de ruta secuencial |
| La dirección está comprometida con el cambio | El modelo requiere liderazgo ejecutivo activo |

**Cómo integrar:** Usar los 8 pasos como lista de verificación para asegurar que todas las dimensiones del cambio se abordan. Complementar con experimentos de Lean Change para la ejecución.

### Escenario 2: Transformación ágil en una organización con experiencia ágil

**Modelo recomendado: [[02-lean-change-framework|Lean Change]]**

| Situación | Justificación |
|-----------|--------------|
| La organización ya tiene experiencia con Scrum/Agile | Lean Change se integra naturalmente con prácticas ágiles |
| Se necesita co-creación con las personas afectadas | Lean Change es fundamentalmente co-creativo |
| Hay alta complejidad y volatilidad | El ciclo iterativo maneja la incertidumbre |
| La dirección está dispuesta a ceder control | Lean Change requiere confianza en las personas |

**Cómo integrar:** Usar el ciclo de Hallazgos → Plan → Ejecución como框架 principal, con Kotter como lista de verificación de que se abordan todas las dimensiones.

### Escenario 3: Desarrollo de capacidades de cambio sostenibles

**Modelo recomendado: [[03-change-agility-model|Change Agility]]**

| Situación | Justificación |
|-----------|--------------|
| La organización necesita mantener agilidad a largo plazo | Change Agility se centra en capacidades sostenibles |
| Los gerentes de nivel intermedio necesitan desarrollo | Los 12 principios guían la práctica de gestión |
| Se busca un cambio cultural profundo | El aprendizaje continuo es fundamental |
| La organización está en un entorno de alta volatilidad | La visión y adaptabilidad son esenciales |

**Cómo integrar:** Usar los dominios de Change Agility como framework de desarrollo de capacidades, con Lean Change para la ejecución de transformaciones específicas.

### Escenario 4: Diagnóstico de resistencia individual al cambio

**Modelo recomendado: [[04-adkar-model|ADKAR]]**

| Situación | Justificación |
|-----------|--------------|
| Las personas individuales están resistiendo el cambio | ADKAR diagnostica en qué etapa están estancadas |
| Se necesita un lenguaje simple para el cambio | ADKAR es fácil de comunicar |
| El cambio requiere nuevas habilidades específicas | ADKAR aborda Knowledge y Ability directamente |
| Se busca medir el progreso individual | ADKAR permite evaluar cada fase |

**Cómo integrar:** Usar ADKAR como herramienta de diagnóstico individual, con Lean Change para la ejecución organizacional y Change Agility para el desarrollo de capacidades.

## Integración de modelos

### Modelo de integración propuesto

En la práctica, los modelos no son mutuamente excluyentes. Little recomienda usar una variedad de métodos y prácticas para maximizar la probabilidad de éxito:

```
┌─────────────────────────────────────────────────────────┐
│                    NIVEL ESTRATÉGICO                     │
│                                                         │
│  Kotter (8 pasos) ←→ Change Agility (4 dominios)       │
│  ─────────────────────────────────────────────────────  │
│  • Urgencia y visión           • Capacidades sostenibles │
│  • Coalición y comunicación    • Principios de gestión   │
│  • Consolidación cultural      • Aprendizaje continuo   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                    NIVEL TÁCTICO                         │
│                                                         │
│  Lean Change (ciclo iterativo)                          │
│  ─────────────────────────────────────────────────────  │
│  • Hallazgos (co-creación)                              │
│  • Plan de Experiencia de Cambio                        │
│  • Ejecución de Experimentos                            │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                    NIVEL INDIVIDUAL                       │
│                                                         │
│  ADKAR (5 etapas)                                       │
│  ─────────────────────────────────────────────────────  │
│  • Awareness → Desire → Knowledge → Ability → Reinforce │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Cómo se complementan los modelos

| Necesidad | Modelo primario | Modelo complementario |
|-----------|-----------------|----------------------|
| Establecer dirección estratégica | Kotter (pasos 1-4) | Change Agility (Visión) |
| Involucrar a las personas | Lean Change (Hallazgos) | ADKAR (Desire) |
| Ejecutar el cambio | Lean Change (Ejecución) | ADKAR (Knowledge/Ability) |
| Manejar resistencia | ADKAR (diagnóstico) | Lean Change (experimentos) |
| Desarrollar capacidades | Change Agility (aprendizaje) | Kotter (consolidación) |
| Medir progreso | Lean Change (métricas) | ADKAR (Reinforcement) |
| Anclar en cultura | Kotter (paso 8) | Change Agility (aprendizaje continuo) |

## Ejemplo de aplicación integrada: La Comisión

La Comisión de gobierno de Ontario, Canadá, ilustra cómo se pueden usar múltiples modelos:

| Modelo usado | Acción | Resultado |
|-------------|--------|-----------|
| **Lean Change** | Ciclo de Hallazgos → Plan → Ejecución | 44 hallazgos, 69 opciones, experimentos de cambio |
| **ADKAR** | Diagnóstico individual de las personas | Identificación de quién necesitaba qué tipo de apoyo |
| **Kotter** | Verificación de los 8 pasos | Asegurar que se abordaron urgencia, visión, comunicación, etc. |
| **Change Agility** | Desarrollo de coaches internos | Formación de 8 coaches de primer nivel |

## Conclusión

No existe un modelo "perfecto" para el cambio. La efectividad depende de:

1. **El contexto organizacional**: Cultura, estructura, historia de cambios anteriores
2. **La naturaleza del cambio**: Complejidad, alcance, urgencia
3. **Las personas**: Nivel de madurez, experiencia previa, disposición
4. **El liderazgo**: Compromiso, estilos, capacidad

La recomendación实践 es usar una combinación de modelos, adaptados al contexto específico. Como dice Little:

> "No hay una receta para hacer que el cambio funcione. Y de hecho, una receta para hacer que el cambio funcione es en realidad una receta para que no funcione."
